HBase 数据模型

​ 在 HBase 中，数据模型同样是由表组成的，各个表中又包含数据行和列，在这些表中存储了 HBase 数据。

  • 术语

    表（Table）

    HBase 会将数据组织进一张张的表里面，一个 HBase 表由多行组成。

    行（Row）

    HBase 中的一行包含一个行键和一个或多个与其相关的值的列。在存储行时，行按字母顺序排序。由于这个原因，行键的设计非常重要。目标是以相关行相互靠近的方式存储数据。常用的行键模式是网站域。如果你的行键是域名，则你可能应该将它们存储在相反的位置（org.apache.www，org.apache.mail，org.apache.jira）。这样，表中的所有 Apache 域都彼此靠近，而不是根据子域的第一个字母分布。

    列（Column）

    HBase 中的列由一个列族和一个列限定符组成，它们由:（冒号）字符分隔。

    列族（Column Family）

    出于性能原因，列族在物理上共同存在一组列和它们的值。在 HBase 中每个列族都有一组存储属性，例如其值是否应缓存在内存中，数据如何压缩或其行编码是如何编码的等等。表中的每一行都有相同的列族，但给定的行可能不会在给定的列族中存储任何内容。

    列族一旦确定后，就不能轻易修改，因为它会影响到 HBase 真实的物理存储结构，但是列族中的列标识(Column Qualifier)以及其对应的值可以动态增删。

    列限定符（Column Qualifier）

    列限定符被添加到列族中，以提供给定数据段的索引。鉴于列族的content，列限定符可能是content:html，而另一个可能是content:pdf。虽然列族在创建表时是固定的，但列限定符是可变的，并且在行之间可能差别很大。

    单元格（Cell）

    单元格是行、列族和列限定符的组合，并且包含值和时间戳，它表示值的版本。

    时间戳（Timestamp）

    时间戳与每个值一起编写，并且是给定版本的值的标识符。默认情况下，时间戳表示写入数据时 RegionServer 上的时间，但可以在将数据放入单元格时指定不同的时间戳值。

服务器流量控制策略-RateLimit

发表于 2018-09-19 | 分类于 Java

应用背景

由于业务系统的负载能力有限，为了防止预期外的请求对系统造成过大的压力而拖垮系统（即大流量），进行的一种流量控制手段

常见策略

常见的服务器流量控制策略有：

  • 分流

    待补充

  • 降级

    待补充

  • 限流

    虽然降低了服务的访问频率和并发量，也因此换来了服务的稳定和高可用性。

    常见的限流方式：

    1. Nginx前端限流

      按照一定的逻辑如账号，IP，系统调用逻辑等在Nginx层做限流

    2. 业务应用系统限流

      • 客户端
      • 服务端

    3. 数据库限流

      红线区，需要力保

常见的限流算法

常用的限流算法有：漏桶算法和令牌桶算法，下面分别进行介绍。

  • 漏桶算法（Leaky Bucket）

    算法的思路很简单，即水(请求)先进入到漏桶里，漏桶以一定的速度出水(接口的处理响应速率)，当水流入速度过大会直接溢出(访问频率超过接口响应速率)，然后就拒绝请求。

    可以看出，漏桶算法可以强行限制数据的传输速率。

  • 令牌桶算法（Token Bucket）

    算法的思路类似漏桶算法：系统按照固定的速率向桶中放入令牌（Token）直到桶中的令牌满了为止，业务处理则在每次响应请求之前先从桶中获取令牌，拿到令牌的请求可以成功响应，拿不到令牌的则阻塞等待或直接返回失败（拒绝服务）。

两种算法的区别

根据特点，漏桶算法能够强行限制数据的传输速率，令牌桶算法除能限制数据的传输速率外，还可以根据令牌放入的速率调整来应对流量的突发。

  2. 重点说明

    请求的许可数从来不会影响到请求本身的限制（调用acquire(1) 和调用acquire(1000) 将得到相同的限制效果，如果存在这样的调用的话），但会影响下一次请求的限制，也就是说，如果一个高开销的任务抵达一个空闲的RateLimiter，它会被马上许可，但是下一个请求会经历额外的限制，从而来偿付高开销任务。

    注意：RateLimiter 并不提供公平性的保证。

返回值类型 方法和描述
double acquire()
从RateLimiter获取一个令牌，该方法会阻塞到令牌获取成功

从RateLimiter获取指定数量的令牌，该方法会阻塞直到令牌获取成功

double getRate()
返回RateLimiter配置的稳定吞吐率（即permits per seconds）
void setRate(double permitsPerSecond)

boolean tryAcquire()

尝试在超时时间内获取指定数量的令牌，否则返回false（非阻塞）
