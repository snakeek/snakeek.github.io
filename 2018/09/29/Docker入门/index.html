下面描述下 docker run alpine ls -l 命令都进行了哪些操作：

  1. Docker client 与 Docker daemon进行交互

  2. Docker daemon检查本地是否有可用的镜像（image），如果没有，则从Docker Store进行下载

    （之前已经通过 docker pull alpine 命令进行了下载，故本次操作不需要）

  3. Docker daemon基于镜像创建容器（即 docker run alpine ），并在容器中执行命令（即 ls -l ）

  4. Docker daemon将命令的执行结果返回给Docker client

这里简单介绍下命令中涉及到的参数

  • -d 创建一个后台运行的容器，并返回容器ID
  • -P 发布容器端口为主机上任意端口
  • -e 通过环境变量的方式将参数传递给容器
  • –name 给容器指定一个名字
  • AUTHOR 环境变量名，其值则是对应的环境变量值

名词释义

  • Images（镜像）

    用来创建容器的应用的文件系统和相关配置信息

    如果想要查看某个image的更多信息，可以使用命令

  • Containers

    容器是真正运行Docker image的实体

    一个容器包含了应用和其相关依赖，并和其他容器共享kernel，并且在主机的用户空间中各自独立运动

    可以使用下述命令查看正在运行中的容器

  • Docker daemon

    在主机执行的后台服务，管理创建，运行，分配Docker容器

  • Docker client

    允许用户和Docker daemon交互的命令行工具

  • Docker store

    Docker image的仓库，可以用到找到可靠的，企业级的容器，插件等
